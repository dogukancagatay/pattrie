    pattrie.Pattrie

    A longest-prefix-match (LPM) IP prefix trie.

    Stores arbitrary Python values keyed by IP prefixes. Lookups return the value associated with the longest matching prefix.

    Parameters:

    Name Type Description Default
    maxbits

    Maximum prefix length accepted (1–32 for IPv4, 1–128 for IPv6). Defaults to 32.

    		 required
    family

    Address family — socket.AF_INET (default) or socket.AF_INET6.

    		 required

    Raises:

    Type Description
    ValueError

    If family is not AF_INET or AF_INET6, or if maxbits is out of range for the chosen family.
    Example 
    import pattrie

t = pattrie.Pattrie()
t["10.0.0.0/8"] = "rfc1918"
t["10.1.0.0/16"] = "internal"

t["10.1.2.3"]  # → "internal"  (longest match wins)
t["10.2.0.1"]  # → "rfc1918"

    __setitem__(key: NetworkKey, value: object) -> None

    Insert or replace a prefix using t[prefix] = value.

    Equivalent to insert(prefix, value). Host bits are zeroed before insertion ("10.1.2.3/8" is stored as "10.0.0.0/8").

    Raises:

    Type Description
    ValueError

    If the trie is frozen or the prefix is invalid.

    		 required

    Returns:

    Type Description
    object

    The value for the longest matching prefix.

    Raises:

    Type Description
    KeyError

    If no prefix in the trie covers key.

    __delitem__(key: NetworkKey) -> None

    Delete a prefix using del t[prefix].

    Requires an exact match. Host bits are zeroed before lookup.

    Raises:

    Type Description
    KeyError

    If the prefix is not in the trie.
    ValueError

    If the trie is frozen or the prefix is invalid.

    __contains__(key: AddressKey) -> bool

    Return True if any prefix in the trie covers key (LPM semantics).

    Does not raise on invalid or unrecognised keys — returns False instead.

    __len__() -> int

    Return the number of prefixes stored in the trie.

    Iterate over all stored prefixes as CIDR strings.

    get_many(keys: Sequence[AddressKey], default: object = None) -> list[object]

    Look up multiple keys in one call, returning a list of values.

    Each key is resolved by longest-prefix match; misses return default. When the trie is frozen, all trie traversals run without the GIL, enabling true parallel use from multiple threads.

    Parameters:

    Name Type Description Default
    keys Sequence[AddressKey]

    A sequence of IP addresses or network prefixes.

    		 required
    default object

    Value to return for each miss. Defaults to None.

    		 None

    Returns:

    Type Description
    list[object]

    A list of matched values (or default) in the same order as
    list[object]

    keys.

    Return the matching prefix for a longest-prefix-match lookup.

    Unlike __getitem__, returns the prefix string (e.g. "10.0.0.0/8") rather than the stored value.

    Parameters:

    Name Type Description Default
    key AddressKey

    An IP address or network prefix.

    		 required

    Returns:

    Type Description
    str | None

    The matched prefix as a CIDR string, or None on a miss.

    has_key(key: NetworkKey) -> bool

    Return True if key is an exact prefix in the trie.

    Unlike __contains__, this requires an exact match — no longest-prefix semantics.

    Parameters:

    Name Type Description Default
    key NetworkKey

    A network prefix (string, IPv4Network, or IPv6Network).

    		 required

    Returns:

    Type Description
    bool

    True if the exact prefix exists, False otherwise.

    delete(key: NetworkKey) -> None

    Delete an exact prefix from the trie.

    Equivalent to del t[prefix]. Host bits are zeroed before lookup.

    Raises:

    Type Description
    KeyError

    If the prefix is not in the trie.
    ValueError

    If the trie is frozen or the prefix is invalid.

    keys() -> list[str]

    Return a list of all stored prefixes as CIDR strings.

    values() -> list[object]

    Return a list of all stored values in trie traversal order.

    Order matches keys() — the i-th value corresponds to the i-th key.

    items() -> list[tuple[str, object]]

    Return a list of (prefix, value) pairs in trie traversal order.

    Each element is a two-tuple of the CIDR prefix string and its associated value, in the same order as keys().

    children(prefix: NetworkKey) -> list[str]

    Return all prefixes in the trie more specific than prefix.

    Uses longest-prefix containment: any stored prefix whose address range is entirely within prefix is included at any depth. The queried prefix itself is never included in the result. Returns [] when no descendants exist or the trie is empty.

    Parameters:

    Name Type Description Default
    prefix NetworkKey

    The parent prefix to query (CIDR string, IPv4Network, or IPv6Network). Host bits are zeroed before lookup.

    		 required

    Returns:

    Type Description
    list[str]

    A list of CIDR strings for all stored prefixes contained within
    list[str]

    prefix. Order is lexicographic (prefix-trie traversal order).

    Raises:

    Type Description
    ValueError

    If prefix belongs to the wrong address family or is malformed.
    Example 
    t = Pattrie()
t["10.0.0.0/8"] = "a"
t["10.1.0.0/16"] = "b"
t["10.1.1.0/24"] = "c"
t["10.2.0.0/16"] = "d"

t.children("10.0.0.0/8")
# → ["10.1.0.0/16", "10.1.1.0/24", "10.2.0.0/16"]

t.children("10.0.0.0/9")   # not stored — still returns descendants
# → ["10.1.0.0/16", "10.1.1.0/24"]

    parent(prefix: NetworkKey) -> str | None

    Return the closest covering prefix for prefix.

    Returns the longest stored prefix that contains prefix but is not prefix itself — i.e. the immediate ancestor in the stored prefix hierarchy. Returns None if no covering prefix exists.

    Parameters:

    Name Type Description Default
    prefix NetworkKey

    The prefix to query (CIDR string, IPv4Network, or IPv6Network). Host bits are zeroed before lookup.

    		 required

    Returns:

    Type Description
    str | None

    The CIDR string of the longest stored prefix that covers
    str | None

    prefix, or None if no such prefix exists.

    Raises:

    Type Description
    ValueError

    If prefix belongs to the wrong address family or is malformed.
    Example 
    t = Pattrie()
t["10.0.0.0/8"] = "a"
t["10.1.0.0/16"] = "b"
t["10.1.1.0/24"] = "c"

t.parent("10.1.1.0/24")   # → "10.1.0.0/16"
t.parent("10.1.0.0/16")   # → "10.0.0.0/8"
t.parent("10.0.0.0/8")    # → None

t.parent("10.2.0.0/16")   # → "10.0.0.0/8"  (not stored — still works)

    freeze() -> None

    Make the trie read-only and enable GIL-free concurrent reads.

    After freezing, __getitem__, get, get_many, and __contains__ all release the GIL, allowing true parallel lookups from multiple threads. Any mutation attempt raises ValueError.

    Use thaw() to restore mutability.

    thaw() -> None

    Restore mutability after freeze().

    Read operations will re-acquire the GIL. Mutations are allowed again.
